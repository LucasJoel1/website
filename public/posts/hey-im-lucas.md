Right now, I am working on programming a calendar backend called Tally.  Tally performs the basic functionality of a calendar app such as adding, creating and deleting events (I know, *revolutionary*), but also uses something called repetition patterns to create simple and complex event repetitions.  I also have plans on adding auto-inserting events within set time period, an HTTP API for interacting with the backend, a cli, *maybe* gui, and a plugin system!  If you find this interesting, I will be open sourcing the current progress soon and continue to write about it here.

Another project on the back burner is a Minecraft panel written in go called openmc-panel.  This is to be a fully featured, lightweight minecraft panel using go for the backend api, and react for the frontend.  The main goal of this project is to create a FOSS minecraft panel that you could run with very minimal overhead.  From personal testing, with 10+ users logged into the panel at once and under load, the memory spikes at about 50mb and stays at 10mb< idle with an unnoticable cpu usage (task manager said 0.1% idle and max 0.5% under load so take that as you will).
